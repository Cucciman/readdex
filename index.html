How to add & customise sources ▼

Adding your own feeds

Everything is configured in one place: the SOURCES array at the top of the <script> block in index.html. No build step, no terminal — just edit the file and reload.

Step 01

Find the RSS URL for your source

Most publications have a public RSS feed. Common patterns — try these first:

  • Substack https://yourpublication.substack.com/feed
  • WordPress https://yoursite.com/feed
  • Ghost https://yoursite.com/rss
  • BBC sections https://feeds.bbci.co.uk/news/business/rss.xml
  • TechCrunch https://techcrunch.com/feed/
  • The Guardian https://www.theguardian.com/business/rss

Not sure if a site has RSS? Add /feed or /rss to the homepage URL and see if it loads XML. You can also paste any URL into RSS Validator to confirm it works.

Step 02

Wrap it through the rss2json proxy

Browsers block direct RSS requests (CORS). The dashboard uses rss2json.com as a free proxy. Just prepend this to any RSS URL:

https://api.rss2json.com/v1/api.json?rss_url=YOUR_FEED_URL

Example — AC Notebook on Substack:

https://api.rss2json.com/v1/api.json?rss_url=https://acnotebook.substack.com/feed
Free tier: 10,000 requests/day — plenty for personal use. If you hit the limit, register at rss2json.com for a free API key and append &api_key=YOUR_KEY to each URL.
Step 03

Add one object to SOURCES[]

Open index.html, find const SOURCES = [, and paste a new entry. Save and reload — done.

{
  id:       "my-source",     // unique, no spaces
  label:    "Source Name",   // shown in card header
  category: "Markets",       // tab it appears under
  color:    "#c8102e",       // dot color in the card header
  rssUrl:   "https://api.rss2json.com/v1/api.json?rss_url=YOUR_FEED_URL"
},

Fields:

  • id — unique key, no spaces (e.g. "bbc-tech")
  • label — name shown in the card header
  • category — reuse an existing one to add to that tab, or write a new one to create a new tab
  • color — hex color for the dot in the card header
  • rssUrl — the proxied feed URL from step 2
Step 04 — Optional

Deploy to GitHub Pages

This is a single static file — no server needed.

  • 1. Create a GitHub repository and upload index.html to the root
  • 2. Go to Settings → Pages → Source → Deploy from branch → main / root
  • 3. Live at https://yourusername.github.io/reponame within ~30 seconds
  • 4. To add sources later: edit index.html directly in GitHub's web editor and commit
Troubleshooting

Feed shows "unavailable"

Some sources block automated access — common with paywalled sites (FT, WSJ, Bloomberg). In that case, look for an alternative open feed for the same source, or use a different publication in the same category. You can test any RSS URL at the validator below before adding it.
